## Overview

Regularized least squares is a supervised learning technique that predicts student success by learning patterns from labeled training data. The model achieves **80.7% accuracy** and **100% precision** in identifying students who will perform above the median.

<Info>
Regularization with **λ=0.01** prevents overfitting while maintaining high predictive accuracy across 11 engineered features.
</Info>

## Mathematical Foundation

The regularized least squares solution minimizes the following objective function:

```
min ||Xβ - y||² + λ||β||²
 β
```

Where:
- `X` ∈ ℝⁿˣᵖ is the feature matrix (n samples, p features)
- `β` ∈ ℝᵖ is the coefficient vector to learn
- `y` ∈ ℝⁿ is the target vector (binary: 0 or 1)
- `λ` is the regularization parameter

### Closed-Form Solution

The optimal coefficient vector is computed using:

```
β = (X'X + λI)⁻¹ X'y
```

<Accordion title="Understanding the Formula">
- **X'X** - Feature covariance matrix
- **λI** - Regularization term (ridge penalty)
- **I** - Identity matrix of size p×p
- **(·)⁻¹** - Matrix inverse
- **X'y** - Feature-target correlation vector

The regularization term `λI` is added to the diagonal of X'X to:
1. Ensure the matrix is invertible
2. Penalize large coefficient values
3. Prevent overfitting to training data
</Accordion>

## Implementation

The model is implemented in Julia with clean, efficient code:

### Core Algorithm

```julia
using LinearAlgebra

# Regularized least squares model
function regularized_ls(X, y, λ)
    n, p = size(X)
    β = (X'X + λ*I(p)) \ (X'y)
    return β
end
```

<Note>
The `\` operator in Julia efficiently solves the linear system without explicitly computing the inverse, providing numerical stability.
</Note>

### Prediction Function

```julia
# Predicts the value of y given X and β
function predict(X, β)
    return X * β
end
```

Predictions are continuous values between 0 and 1, representing the probability of success.

### Model Evaluation

```julia
# Evaluates the accuracy of the model by comparing predicted to true values
function evaluate_model(y_true, y_pred)
    predictions = y_pred .>= 0.5
    accuracy = mean(predictions .== y_true)
    return accuracy
end
```

<Info>
A threshold of 0.5 converts continuous predictions to binary classifications (0 = below median, 1 = above median performance).
</Info>

```julia
function prepare_features(df)

    question_df = df[df.content_type_id .== 0, :]
    sort!(question_df, [:user_id, :timestamp])
    
    # Aggregate per-user features
    user_features = combine(groupby(question_df, :user_id)) do user_data
        # [Feature computation logic]
    end
    
    # Create feature matrix
    feature_cols = [:mean_acc, :std_acc, :num_questions, :mean_time_between, 
                   :std_time_between, :recent_acc, :performance_trend, 
                   :mean_container_acc, :std_container_acc, :explanation_impact, 
                   :time_acc_correlation]
    
    X = Matrix{Float64}(user_features[:, feature_cols])
    replace!(X, NaN=>0.0, Inf=>0.0, -Inf=>0.0)
    
    # Normalization
    X = (X .- mean(X, dims=1)) ./ std(X, dims=1)
    
    # Target: above median = 1, below = 0
    y = float.(user_features.mean_acc .>= median(user_features.mean_acc))
    
    return X, y, user_features, feature_cols
end
```

## Feature Normalization

All features are normalized using z-score standardization:

```julia
# Normalization: (X - μ) / σ
X = (X .- mean(X, dims=1)) ./ std(X, dims=1)
```

<Note>
Normalization ensures that features with different scales (e.g., accuracy 0-1 vs. timestamps in milliseconds) contribute equally to the model.
</Note>

## Train-Test Split

The data is split into training (80%) and testing (20%) sets with random shuffling:

```julia
# Splits the data into training and testing sets
function split_data(X, y, train_ratio=0.8)
    n = size(X, 1)
    shuffle_idx = randperm(n)
    train_size = floor(Int, train_ratio * n)
    
    X_train = X[shuffle_idx[1:train_size], :]
    y_train = y[shuffle_idx[1:train_size]]
    X_test = X[shuffle_idx[(train_size+1):end], :]
    y_test = y[shuffle_idx[(train_size+1):end]]
    
    return X_train, y_train, X_test, y_test
end
```

## Performance Metrics

The model is evaluated using a comprehensive confusion matrix:

```julia
function print_confusion_matrix_table(X, y, β)
    y_pred_continuous = predict(X, β)
    y_pred_binary = y_pred_continuous .>= 0.5
    
    # Compute confusion matrix components
    tp = sum((y .== 1) .& (y_pred_binary .== 1))  # True Positives
    fp = sum((y .== 0) .& (y_pred_binary .== 1))  # False Positives
    tn = sum((y .== 0) .& (y_pred_binary .== 0))  # True Negatives
    fn = sum((y .== 1) .& (y_pred_binary .== 0))  # False Negatives
    
    total = tp + fp + tn + fn
    
    # Calculate metrics
    accuracy = (tp + tn) / total
    precision = tp / (tp + fp)
    
    println("Accuracy:  $(round(accuracy * 100, digits=2))%")
    println("Precision: $(round(precision * 100, digits=2))%")
end
```

## Why Regularization?

Regularization addresses several challenges:

### Without Regularization (λ=0)
- X'X may be singular or near-singular
- Large coefficient values lead to overfitting
- Poor generalization to new students

### With Regularization (λ=0.01)
- Ensures X'X + λI is invertible
- Constrains coefficient magnitudes
- Better generalization: 80.7% accuracy
- Prevents overfitting to training noise

<Note>
The small value **λ=0.01** provides just enough regularization to stabilize the solution without underfitting the training data.
</Note>
